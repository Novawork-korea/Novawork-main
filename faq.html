FAQ

서비스 선택 전에 헷갈리는 부분을 먼저 정리했습니다.

각 서비스는 작업 목적, 준비 자료, 수정 기준, 제외 범위가 다릅니다. 문의 전 자주 묻는 내용을 기준으로 확인해 보세요.

서비스 선택비용·일정수정·오류 확인

PROCESS

기본 진행 방식

어떤 서비스로 접수해야 할지 모르겠으면 어떻게 하나요?

정확한 서비스명을 고르지 않아도 됩니다. 현재 문제, 원하는 결과, 운영 중인 사이트 URL, 참고 자료를 남겨주시면 홈페이지 제작, 랜딩페이지, 웹시스템, 수정, GA4/GTM, AI 챗봇, 데이터 수집 중 맞는 범위로 정리해드립니다.

기획이 아직 정리되지 않았어도 접수할 수 있나요?

가능합니다. 처음에는 목표, 대상 고객, 필요한 화면, 문의 동선부터 정리합니다. 다만 실제 착수 전에는 화면 수, 기능 범위, 준비 자료, 완료 기준을 확정해야 일정과 비용이 안정적입니다.

작업은 어떤 순서로 진행되나요?

접수 내용을 확인한 뒤 범위와 가능 여부를 먼저 안내합니다. 이후 자료 확인, 견적·일정 확정, 제작 또는 수정, 중간 확인, 최종 검수, 배포 또는 파일 전달 순서로 진행합니다.

저렴한 템플릿 제작과 NOVAWORK 작업은 무엇이 다른가요?

템플릿 제작은 빠르게 공개하는 데 유리하지만 구조 변경과 추적 세팅에 제한이 있을 수 있습니다. NOVAWORK는 작업 범위에 따라 직접 코딩, 기존 사이트 수정, 문의 동선, GA4/GTM 이벤트, 운영 기준까지 구분해 안내합니다.

제작 후 제가 직접 수정할 수 있나요?

정적 코드 기반 결과물은 직접 수정이 가능하지만 HTML/CSS 이해가 필요합니다. 직접 수정이 필요한 경우 작업 전 운영 방식에 맞춰 안내드립니다.

소스 파일은 받을 수 있나요?

직접 제작한 정적 페이지와 합의된 산출물은 전달 가능합니다. 외부 빌더, 유료 테마, 플러그인, API 계정은 각 서비스 정책에 따라 전달 범위가 달라집니다.

기존 사이트를 보고 개선점만 진단받을 수 있나요?

가능합니다. 모바일 깨짐, 문의 동선, CTA, 속도, GA4/GTM, SEO 기초 항목을 기준으로 점검 범위를 나눌 수 있습니다.

디자인 시안도 먼저 볼 수 있나요?

작업 범위에 따라 와이어프레임, 섹션 구조안, 이미지 시안, 테스트 링크 중 적합한 방식으로 공유합니다.

광고 성과나 검색 순위를 보장하나요?

광고 성과와 검색 순위는 보장하지 않습니다. 다만 문의 동선, 페이지 구조, 기본 SEO, 전환 추적처럼 성과를 확인하고 개선하기 위한 기반은 작업 범위에 따라 구축할 수 있습니다.

작업 전에 꼭 준비해야 하는 자료가 있나요?

현재 사이트 URL, 원하는 결과, 참고 사이트, 로고와 이미지, 주요 문구, 도메인/호스팅 정보, 측정하고 싶은 버튼 목록이 있으면 견적과 일정 확인이 빨라집니다.

ESTIMATE

비용 / 견적

견적은 어떤 기준으로 달라지나요?

페이지 수, 섹션 수, 입력폼·로그인·관리자 기능, 외부 서비스 연동, 데이터 구조, 자료 준비 상태, 긴급 여부에 따라 달라집니다. 단순 수정과 신규 웹시스템 개발은 산정 기준이 다릅니다.

정확한 견적을 받으려면 무엇을 보내야 하나요?

현재 사이트 또는 참고 사이트 URL, 필요한 기능, 원하는 페이지 수, 준비된 로고·이미지·문구, 희망 일정, 예산 범위를 보내주시면 견적 범위를 더 빠르게 좁힐 수 있습니다.

예산이 정해져 있지 않아도 문의할 수 있나요?

가능합니다. 예산이 미정이면 필수 기능과 선택 기능을 나누고, 우선 제작해야 할 범위부터 제안합니다. 단, 확정 견적은 실제 작업 범위가 정리된 뒤 안내드립니다.

SCHEDULE

제작 기간

작업 기간은 보통 얼마나 걸리나요?

간단한 문구·링크·반응형 수정은 범위 확인 후 빠르게 처리할 수 있습니다. 홈페이지·랜딩페이지는 화면 수와 자료 준비 상태에 따라 달라지며, React·Firebase 웹시스템은 데이터 구조와 관리자 기능 확정 후 일정을 안내합니다.

급한 작업도 가능한가요?

가능 여부를 먼저 확인합니다. 긴급 작업은 범위를 작게 나누어 핵심 문제부터 처리하는 방식이 안정적입니다. 기존 코드 구조나 계정 권한 확인이 필요한 경우 즉시 착수가 어려울 수 있습니다.

자료 전달이 늦어지면 일정은 어떻게 되나요?

필수 자료가 늦어지면 제작 일정도 함께 조정될 수 있습니다. 일정이 중요한 작업은 착수 전에 필요한 문구, 이미지, 계정 권한, 참고 자료를 먼저 정리하는 것을 권장합니다.

WEBSITE

웹사이트 제작

홈페이지 제작에는 무엇이 포함되나요?

메인 메시지, 서비스 소개 섹션, 문의 버튼, PC·모바일 반응형, 기본 메타태그, 공유 이미지 설정, 폼 또는 외부 문의 동선 연결처럼 방문자가 정보를 보고 문의하기까지 필요한 흐름을 구성합니다.

디자인 시안 없이도 제작이 가능한가요?

가능합니다. 참고 사이트, 브랜드 색감, 필요한 섹션, 문구 방향을 기준으로 화면 구조를 잡습니다. 이미 보유한 Figma, 이미지 시안, 브랜드 가이드가 있다면 그 기준에 맞춰 구현할 수 있습니다.

기존 사이트를 유지하면서 일부만 개선할 수 있나요?

가능합니다. 다만 기존 코드 구조, 사용 중인 빌더·테마·플러그인, 배포 방식에 따라 수정 가능 범위가 달라집니다. 먼저 파일 구조와 권한 확인이 필요합니다.

LANDING

랜딩페이지 제작

랜딩페이지는 홈페이지와 무엇이 다른가요?

랜딩페이지는 특정 광고, 상품, 이벤트, 상담 접수처럼 하나의 전환 목표에 집중합니다. 메뉴가 많은 일반 홈페이지보다 메시지, 혜택, 신뢰 요소, 문의 버튼 흐름을 더 압축적으로 설계합니다.

랜딩페이지에 문의폼이나 카카오톡 버튼을 넣을 수 있나요?

가능합니다. 문의폼, 전화, 이메일, 카카오톡 채널, 외부 예약 링크 등 필요한 전환 동선을 연결할 수 있습니다. 폼 저장 방식이나 수신 방식은 착수 전에 확인합니다.

광고용 랜딩페이지 전환 추적도 같이 가능한가요?

작업 범위에 포함하면 GA4/GTM 이벤트, 전화·카카오톡·폼 제출 클릭 추적, 광고 계정 연동 준비를 함께 점검할 수 있습니다. 광고 성과나 검색 순위 자체를 보장하지는 않습니다.

MAINTENANCE

유지관리

유지보수만 따로 맡길 수 있나요?

가능합니다. 기존 사이트 구조를 확인한 뒤 정기 유지관리와 단발성 수정 중 적합한 방식으로 안내합니다. 외부 빌더나 테마 기반 사이트는 수정 가능 범위 확인이 먼저 필요합니다.

유지관리에는 어떤 작업이 포함될 수 있나요?

문구·이미지 교체, 버튼 링크 수정, 섹션 추가, 간단한 스타일 조정, 반응형 깨짐 확인, 추적 코드 점검, 문의 동선 확인 등이 포함될 수 있습니다. 신규 기능 개발은 별도 범위입니다.

운영 중 오류가 발생하면 바로 수정할 수 있나요?

사이트 내부 코드나 설정 문제는 확인할 수 있습니다. 다만 호스팅 장애, 외부 API 장애, 카카오·Google 같은 타사 서비스 장애는 직접 수정이 아니라 원인 분리와 대응 안내 중심으로 진행됩니다.

REVIEW

수정 / 검수

수정 요청은 어디까지 포함되나요?

합의된 작업 범위 안에서 오탈자, 여백, 문구, 이미지 교체, 버튼 링크, 반응형 깨짐 같은 항목은 검수 단계에서 반영합니다. 새로운 페이지나 기능 추가는 별도 범위로 판단합니다.

작업 완료 후 오류 확인 기간이 있나요?

홈페이지 제작, 수정, GA4/GTM, AI 챗봇, 크롤링은 보통 납품 후 7일 이내 작업 범위 내 오류를 확인합니다. React·Firebase 웹시스템은 최종 납품 후 14일 이내 기능 오류 확인을 기준으로 안내합니다.

검수는 어떤 기준으로 하나요?

PC와 모바일 화면, 주요 버튼 링크, 문의폼 제출 흐름, 반응형 깨짐, 콘솔 오류, 이미지 경로, 메타 정보, 합의된 기능 동작을 중심으로 확인합니다. 브라우저 범위는 작업 전 협의합니다.

MATERIALS

자료 전달

작업을 빨리 시작하려면 어떤 자료가 필요하나요?

운영 중인 사이트 URL, 원하는 결과, 참고 사이트, 로고·이미지·문구, 도메인·호스팅·관리자 계정 권한, 문의폼 수신 방식, 측정하려는 버튼 목록을 준비하면 확인 속도가 빨라집니다.

문구를 직접 작성하지 못해도 되나요?

기본 방향과 핵심 정보가 있으면 문구 구조를 함께 정리할 수 있습니다. 다만 업종별 법적 고지, 가격, 정책, 전문 용어의 최종 정확성은 의뢰인이 확인해야 합니다.

이미지나 로고가 없으면 어떻게 하나요?

보유 자료가 없으면 텍스트 중심 구조로 먼저 제작하거나, 사용 가능한 기본 이미지 방향을 제안할 수 있습니다. 유료 이미지 구매, 촬영, 일러스트 제작은 별도 범위입니다.

DOMAIN

도메인 / 호스팅

도메인·호스팅·배포도 같이 진행할 수 있나요?

가능한 범위 안에서 안내할 수 있습니다. 도메인 소유권, 호스팅 계정, DNS, 외부 플랫폼 권한은 의뢰인 계정 기준으로 진행하며 필요한 권한을 먼저 확인해야 합니다.

기존 도메인을 새 사이트에 연결할 수 있나요?

대부분 가능합니다. 다만 현재 도메인이 어디에서 관리되는지, DNS 권한이 있는지, 기존 메일이나 다른 서비스와 연결되어 있는지 확인해야 안전하게 이전할 수 있습니다.

호스팅은 어떤 것을 사용하나요?

정적 홈페이지, React 앱, Firebase 기반 시스템, 기존 호스팅 수정 등 작업 구조에 따라 다릅니다. 유지 비용, 배포 방식, 관리자 접근 권한을 고려해 적합한 방식을 안내합니다.

SEO

SEO / 검색 노출

SEO도 같이 되나요?

기본 메타태그, 제목 구조, 설명 문구, OG 이미지, 사이트맵, robots.txt 같은 기초 항목은 작업 범위에 따라 반영할 수 있습니다. 검색 순위 보장이나 대규모 콘텐츠 SEO 운영은 별도 영역입니다.

네이버나 구글 검색에 바로 노출되나요?

사이트가 공개되고 검색 엔진이 수집할 수 있는 구조라면 노출 준비를 할 수 있습니다. 다만 실제 색인 시점과 검색 순위는 검색 엔진 정책, 콘텐츠 품질, 도메인 상태에 영향을 받습니다.

기존 사이트의 검색 노출이 떨어지지 않게 할 수 있나요?

URL 구조, 제목, 설명, 기존 페이지 삭제 여부, 리디렉션, 사이트맵을 확인해 위험을 줄일 수 있습니다. 다만 개편 후 검색 성과 변동을 완전히 보장할 수는 없습니다.

CONTRACT

결제 / 계약

계약이나 결제는 언제 진행하나요?

작업 범위, 일정, 산출물, 검수 기준을 확인한 뒤 진행합니다. 소규모 수정과 신규 제작은 진행 방식이 다를 수 있으며, 확정 전에는 비용과 포함 범위를 먼저 안내합니다.

작업 중 범위가 늘어나면 어떻게 하나요?

처음 합의한 범위를 기준으로 진행합니다. 새로운 페이지, 기능, 외부 연동, 추가 검수 범위가 생기면 일정과 비용을 별도로 안내한 뒤 반영 여부를 결정합니다.

세금계산서나 증빙 발행이 가능한가요?

거래 방식에 따라 가능한 증빙을 안내합니다. 필요한 증빙 종류, 사업자 정보, 결제 일정을 접수 단계에서 함께 남겨주시면 확인이 빠릅니다.

EXCLUSION

작업 제외 범위

진행이 어려운 작업도 있나요?

불법 수집, 개인정보·민감정보 무단 처리, 캡차·차단 우회, 타사 서비스 정책 위반, 저작권 침해 가능성이 있는 작업, 계정 권한이 확인되지 않는 작업은 진행하지 않습니다.

웹 데이터 크롤링은 모든 사이트가 가능한가요?
